## 3. 鉴权规则

管理员接口统一前缀：

```text
/admin/users
```

所有管理员接口都需要登录态：

```http
Authorization: Bearer <access_token>
```

后端校验逻辑：

- token 无效或过期：返回 `401` 或 `403`
- 当前用户不存在：返回 `401`
- 当前用户已被冻结：返回 `403`
- 当前用户不是管理员：返回 `403`

冻结用户的影响：

- 已冻结用户不能通过 `/auth/login` 登录
- 已冻结用户即使之前有 token，访问依赖当前用户的接口也会被拒绝

## 4. 登录接口

管理员登录仍使用现有普通登录接口。

```http
POST /auth/login
Content-Type: application/json
```

请求示例：

```json
{
  "email": "admin@ainame.local",
  "password": "Admin123456"
}
```

成功响应示例：

```json
{
  "user": {
    "id": 3,
    "username": "admin",
    "email": "admin@ainame.local",
    "free_quota": 999999,
    "is_admin": true,
    "is_frozen": false
  },
  "token": "<access_token>"
}
```

说明：因为 `UserSchema` 新增了 `is_admin` 和 `is_frozen`，普通登录响应里的 `user` 也会带上这两个字段。

## 5. 用户列表

```http
GET /admin/users?page=1&page_size=20&keyword=xxx
Authorization: Bearer <token>
```

Query 参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `page` | int | 否 | 页码，默认 `1`，最小 `1` |
| `page_size` | int | 否 | 每页数量，默认 `20`，范围 `1-100` |
| `keyword` | string | 否 | 按邮箱或用户名模糊搜索 |

成功响应示例：

```json
{
  "items": [
    {
      "id": 3,
      "username": "admin",
      "email": "admin@ainame.local",
      "free_quota": 999999,
      "is_admin": true,
      "is_frozen": false
    }
  ],
  "total": 1,
  "page": 1,
  "page_size": 20
}
```

前端建议：

- 表格字段可以先展示：`id`、`email`、`username`、`free_quota`、`is_admin`、`is_frozen`
- `is_frozen=true` 显示为“已冻结”，否则显示“正常”
- `is_admin=true` 显示为“管理员”，否则显示“普通用户”

## 6. 修改用户信息

```http
PATCH /admin/users/{user_id}
Authorization: Bearer <token>
Content-Type: application/json
```

请求字段均为可选，至少传一个字段。

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `email` | string | 否 | 邮箱，必须合法且不能和其他用户重复 |
| `username` | string | 否 | 用户名，长度 `4-50` |
| `free_quota` | int | 否 | 免费额度，不能小于 `0` |
| `is_admin` | boolean | 否 | 是否管理员 |
| `is_frozen` | boolean | 否 | 是否冻结 |

请求示例：

```json
{
  "username": "newname",
  "free_quota": 10,
  "is_frozen": false
}
```

成功响应示例：

```json
{
  "id": 5,
  "username": "newname",
  "email": "user@example.com",
  "free_quota": 10,
  "is_admin": false,
  "is_frozen": false
}
```

保护规则：

- 不能把当前登录管理员自己的 `is_frozen` 改成 `true`
- 不能把当前登录管理员自己的 `is_admin` 改成 `false`
- 邮箱不能改成其他用户已使用的邮箱

常见错误：

```json
{
  "detail": "没有需要修改的字段"
}
```

```json
{
  "detail": "邮箱已被其他用户使用"
}
```

```json
{
  "detail": "用户不存在"
}
```

## 7. 冻结/解冻用户

```http
POST /admin/users/{user_id}/freeze
Authorization: Bearer <token>
Content-Type: application/json
```

请求示例：冻结用户

```json
{
  "is_frozen": true
}
```

请求示例：解冻用户

```json
{
  "is_frozen": false
}
```

成功响应示例：

```json
{
  "id": 5,
  "username": "tester",
  "email": "user@example.com",
  "free_quota": 3,
  "is_admin": false,
  "is_frozen": true
}
```

保护规则：

- 当前登录管理员不能冻结自己

## 8. 重置密码

```http
POST /admin/users/{user_id}/reset-password
Authorization: Bearer <token>
Content-Type: application/json
```

请求字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `password` | string | 是 | 新密码，长度 `6-50` |

请求示例：

```json
{
  "password": "Newpass123"
}
```

成功响应示例：

```json
{
  "message": "密码已重置"
}
```

说明：

- 后端不会把新密码回显给前端
- 密码会用项目现有 `pwdlib` 哈希逻辑保存到数据库

## 9. 删除用户

```http
DELETE /admin/users/{user_id}
Authorization: Bearer <token>
```

成功响应示例：

```json
{
  "message": "用户已删除"
}
```

保护规则：

- 当前登录管理员不能删除自己

注意：

- 当前实现是物理删除用户
- 如果用户有关联业务记录，数据库外键可能阻止删除，前端需要按后端返回的错误提示处理
- 后续如果需要保留审计记录，建议改成软删除

## 10. 状态码约定

常见状态码：

| 状态码 | 场景 |
| --- | --- |
| `200` | 请求成功 |
| `400` | 请求参数合法性错误，或触发保护规则 |
| `401` | 未登录、token 无效、用户不存在 |
| `403` | token 过期、无管理员权限、账号被冻结 |
| `404` | 用户不存在 |
| `422` | Pydantic 参数校验失败，例如邮箱格式错误、密码太短 |

错误响应通常是：

```json
{
  "detail": "错误原因"
}
```

或 FastAPI/Pydantic 默认校验错误数组。

## 13. 前端对接建议

前端可以先做一个最小管理员用户管理页：

1. 管理员登录页复用现有登录接口
2. 登录成功后保存 token
3. 用户管理表格调用 `GET /admin/users`
4. 表格操作按钮：
   - 编辑：调 `PATCH /admin/users/{user_id}`
   - 冻结/解冻：调 `POST /admin/users/{user_id}/freeze`
   - 重置密码：调 `POST /admin/users/{user_id}/reset-password`
   - 删除：调 `DELETE /admin/users/{user_id}`
5. 前端按钮保护：
   - 当前登录管理员自己的行，不显示“冻结自己”和“删除自己”
   - 重置密码弹窗不要展示明文历史密码，只输入新密码
